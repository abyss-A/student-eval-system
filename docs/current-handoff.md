## 3. 最近完成的业务修订
### 导出与审核交互统一
- 学生端正式导出 Word 当前以 `COUNSELOR_REVIEWED` 为主要开放门槛；代码同时兼容 `FINALIZED / PUBLISHED` 历史状态
- 管理员主表总分改为按评审生效口径实时重算后展示
- Word 基础信息区对长班级名做了定向适配，避免“年级班级”被截断
- 辅导员抽屉 `撤销 / 撤回` 文案统一为 `撤回`
- 辅导员活动表 `证明图片` 改为 `证明材料`
- 管理员抽屉课程表 / 活动表共享语义列名与顺序已统一
- 学生删除被驳回条目改成“待提交删除 -> 再次提交时才送审”
- 管理员测评单页当前为只读查看页，不承担终审测评单职责

### 账号口径调整
- 学生和辅导员账号改为由管理员统一创建，自助注册退出正式业务流程
- 管理员侧新增账号管理页，支持单个创建、Excel 导入、启停、重置初始密码和受限删除
- 登录页不再展示“创建新账户”入口，访问 `/register` 会回到登录页并提示联系管理员
- 初始密码当前统一为 `123456`，用户登录后仍可在账号中心自行修改密码

### 学期管理与排名按学期选择
- 管理员侧新增学期管理页：支持新建学期（默认不立即生效）与切换当前学期
- 切换当前学期前会校验本学期 `SUBMITTED` 待审核数量，未清零时禁止切换并提示数量
- 切换为当前学期时，如目标学期缺少 `scoring_config`，会自动复制上一当前学期配置；若无可复制来源则回退插入默认配置
- 学期管理页支持“评分配置（权重/上限）/编辑名称/删除学期/手动重算”等更多操作：
  - 权重使用小数 `0~1`，并校验五项权重和约等于 1
  - 评分配置新增智育/体育二级占比：
    - 智育：课程平均占比 + 创新活动占比 = 1
    - 体育：大学体育占比 + 体育活动占比（除大学体育） = 1
  - 保存评分配置不自动重算；如要让历史排名按新配置更新，需要手动触发“重算本学期成绩”
  - 删除仅允许“非当前且无 submission”的学期
- 综合排名页学期选择由“手填学期ID”改为“学期名称下拉”，默认当前学期，并可切换查看历史排名

### 三端工作台与账号管理批量操作

## 4. 当前要特别注意的点
### 4.1 导出模板
- 正式模板：`backend/src/main/resources/report-templates/scholarship_form_v1.docx`
- 不要再把调试模板散落在 `backend/` 根目录
- 如果继续调模板，建议按版本化命名

### 4.2 分数口径
- 学生页同时存在预览分和审核分
- 管理员列表按评审生效总分展示
- 管理员查看的是辅导员审核后的正式结果，不对测评单做终审推进
- 修改任何“总分 / 原始分 / 审核分”前，先确认业务口径

### 4.5 评分配置与排名
- 综合排名页展示基于 `submission.total_score`（落库总分），不是实时按配置计算
- 因此修改某学期评分配置后，若要让历史数据生效，需要在学期管理中手动触发“重算本学期成绩”

### 4.3 删除复审
- `DELETE_PENDING_SUBMIT` 是新增内部状态
- 它表示“学生已标记删除，但还没正式送老师复审”
- 只有 `DELETE_REQUESTED` 才表示老师侧真正可见的删除复审

### 4.4 文档与编码
- Markdown / 文本文件统一按 `UTF-8 with BOM`
- 代码文件统一按 `UTF-8`
- 后续在 Windows PowerShell 5.1 下批量写中文文档时，必须显式指定编码
