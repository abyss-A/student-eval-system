# UI 参考与风格口径

## 1. 总体风格
本项目 UI 不走“默认 Element Plus 后台模板”路线，而是追求：
- 紧凑
- 清爽
- 层级清楚
- 业务感强
- 接近“学院工作台”而不是通用 SaaS 模板

不要做成：
- 搜索框很宽
- 按钮很大很空
- 表格层层套娃边框
- 固定列和普通列观感割裂
- 颜色和字号缺乏统一基线

## 2. 已确认的视觉偏好
### 搜索区
- 搜索框要明显偏窄，不要抢布局
- 搜索框一般略宽于筛选下拉即可
- 顶部工具栏留白要克制
- 搜索框外观与筛选下拉统一（同圆角、同边框/焦点态），不使用“胶囊 pill”风格
- 下拉筛选默认占位文案统一中文（如：`全部角色 / 全部状态`），避免出现 `Select`
- 综合排名页学期筛选使用“学期名称下拉”（如：`2026年春季学期`），默认当前学期，不再手填学期ID
- 综合排名页学期切换即自动加载数据，不需要额外“查询”按钮

### 首页（工作台内容）
- 三端“首页”作为默认落地页（登录后及访问 `/student` `/teacher` `/admin` 会重定向到 `/xxx/home`）。
- 顶部标题与固定标签页统一显示为 `首页`；左侧菜单不再提供首页入口（通过固定标签页或点击左上角品牌返回首页）。
- 顶部“工具栏 + 标签栏”采用平铺一体的紧凑吸顶样式（不做两块卡片式大圆角/阴影），整体更薄更稳。
- 左侧菜单默认折叠非核心分组，并在路由命中时自动展开“当前页面所在分组”，保证当前高亮项可见。
- 首页采用“看板化”布局，但做减法：以 Hero 概览 + 核心待办/预览为主，不在首页堆“快捷入口磁贴”（入口统一靠左侧菜单）。
- 1440×900 桌面基准下，首页尽量做到“一屏无纵向滚动”：控制预览条目数量、提示文案单行省略，避免长文本换行把卡片撑高。
- 学生端首页只展示“预览总分”和审核进度，不在首页展开五育分数明细（明细进“综合成绩”查看）。
- 学生端首页待办提醒不提供“反馈入口/未关闭反馈”计数（反馈统一走左侧菜单入口）。
- 辅导员/管理员首页不再展示公告块（公告通过左侧菜单进入）。
- 辅导员/管理员首页支持“一键直达”：点击预览条目跳转列表并自动打开抽屉（`?open=<id>&preset=<filter>`，执行一次后会清理 query）。

### 表格
- 表格是主要信息载体，优先保证对齐、整齐、可读
- 状态列和操作列默认紧凑，且在桌面端固定右侧
- 操作按钮不要太宽太高，文字要完整显示
- 表格内长文本允许通过展开浮层查看：浮层内可换行、最大高度可滚动，不做省略号截断，且不破坏主布局

### 分页
- 分页区保持简洁
- 数字输入居中
- 不要出现奇怪横向滚动条挤压主体内容

### 边框与层级
- 避免双层边框、套娃卡片
- 页面只保留必要的层级：页面主框、数据区边框、弹窗层
- 输入控件只保留一套边框与焦点态，不要叠加浏览器原生样式和组件库样式

## 3. 已确认的交互口径
### 学生端
- 课程和活动均支持自动保存
- 首页展示：本学期测评单状态、审核阶段/进度、预览总分、待办提醒与公告
- 在辅导员未完成整单审核前，学生只能看预览 / 审核状态，不可导出正式 Word
- 被驳回条目在学生修订阶段可修改
- 被驳回条目删除时：
  - 点击删除后先进入“待提交删除”
  - 不立刻送老师复审
  - 只有点“再次提交”才真正进入删除复审
- 删除意图需要可撤回

### 辅导员端
- 审核区按钮要小、紧凑、统一
- 首页展示：阶段分布（未审核/正在审核/待复审/待提交/已提交）+ 待办列表预览 + 待提交管理员统计；支持一键直达自动打开审核抽屉
- 同一语义按钮尽量统一文案
- 当前已确认：`撤销 / 撤回` 在抽屉里统一显示为 `撤回`
- 活动表 `证明图片` 已统一改为 `证明材料`

### 管理员端
- 管理员主要看“辅导员审核后的正式结果”
- 管理员对测评单为只读查看，不提供终审按钮或状态推进操作
- 总分口径按评审生效分，不按学生原始预览分
- 首页展示：当前学期概览（待审核数）、可查看测评单体量、待处理反馈预览、测评单动态预览；支持一键直达自动打开抽屉
- 抽屉里的课程和活动表，公共语义列要统一命名和位置
- 账号管理页延续紧凑搜索区 + 表格主区 + 固定操作列的后台风格
- 账号管理支持批量操作：勾选后可批量启用/停用/重置密码；批量完成提示成功/失败/跳过并自动刷新列表
- 学期管理页用于新建与切换“当前学期”：新建默认不生效；切换前若本学期待审核未清零，会禁止切换并提示数量
- 学期管理页支持“评分配置（权重/上限/智育体育二级占比）/编辑名称/删除学期”等更多操作：权重与二级占比使用小数 `0~1` 且对应合计需为 1；保存配置不自动重算，需手动触发“重算本学期成绩”后排名才按新配置更新；删除仅允许“非当前且无 submission”的学期
- 登录页不再出现自助注册入口，账号开通统一引导为“联系管理员”

## 4. 表头与术语统一口径
### 优先用语
- `原始分`
- `证明材料`
- `审核状态`
- `审核分`
- `审核理由`
- `待提交删除`
- `待删除复审`

### 避免混用
- 不要同一页面一处叫 `自评分`，另一处叫 `原始分`
- 不要同义列在课程表和活动表里前后顺序不同
- 不要一会儿叫 `证明图片`，一会儿叫 `证明材料`

## 5. 导出相关口径
- 正式导出属于“正式材料”，不是草稿预览
- 导出门槛：辅导员提交管理员后才开放
- Word 模板中基础信息区必须能容纳真实班级名，不允许长班级名被截断
- 导出内容默认按正式有效结果理解，避免与页面原始预览值打架

## 6. 后续做新页面时的默认要求
- 默认先保证桌面端表格体验，再做移动端退化
- 有大量数据时优先考虑固定列、列宽、长文本处理
- 改任何列表页前，先确认：
  - 总分 / 分数字段口径
  - 是否要固定右列
  - 按钮是否需要更紧凑
  - 表头术语是否已和其他页统一
